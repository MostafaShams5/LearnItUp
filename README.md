This is my personal cheat sheet and reference guide for Object-Oriented Programming and SOLID principles in Python. 

I wrote this notebook to explain these concepts to myself in plain English, without the confusing academic jargon. 

### What's in the notebook
* **Classes & Objects:** The basics, plus Instance, Class, and Static methods.
* **Encapsulation:** Hiding data and how Python name mangling actually works (`__private`).
* **Inheritance:** Passing down traits, using `super()`, and dealing with multiple inheritance.
* **Polymorphism:** Duck typing, method overriding, and magic/dunder methods (like `__add__`).
* **Abstraction:** Forcing structural rules using `ABC` and `@abstractmethod`.
* **S.O.L.I.D. Principles:** How to write clean code that doesn't break when I need to add new features.

Basically, this is here for me to quickly look up how things work when I forget the exact syntax or logic.
